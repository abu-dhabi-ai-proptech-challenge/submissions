# Submission Guide

## TL;DR

1. Build in your own GitHub repo (public, or private with judge access).
2. Before the deadline, open an Issue here using the **[Project Submission form](../../../issues/new?template=project-submission.yml)**.
3. One submission per team. The Issue timestamp is your submission time — late submissions are not judged.

## Step by step

### 1. Get your project into a repo

- Use your own repo, or generate one from the [`project-template`](https://github.com/abu-dhabi-ai-proptech-challenge/project-template) ("Use this template" button).
- Public is preferred. If private, add the organizer account as a collaborator so judges can review.
- Make sure the README explains how to run it.

### 2. Prepare your links

| Item | Required |
|---|---|
| GitHub repo link | ✅ |
| Demo link (deployed app or 2–3 min video) | ✅ |
| Slides link | Optional, recommended |

A deployed URL (Vercel, Hugging Face Spaces, etc.) is best. A short screen recording is a solid fallback if deployment is fighting you at the deadline.

### 3. Fill in the form

Every field matters, but judges weight two especially:

- **What was built during the hackathon** — distinguish prior boilerplate from today's work. Specific and honest beats long and vague.
- **Setup instructions** — if a judge can't run it, the demo link carries all the weight.

### 4. After submitting

- Organizers apply the `submission` label and your track label, then acknowledge in a comment.
- You may edit the Issue to fix typos or broken links before the deadline. Substantive changes after the deadline are not considered.
- Watch the Issue for judge questions.

## Final checklist

- [ ] Repo opens in an incognito window (or organizer access granted)
- [ ] README explains how to run the project
- [ ] Demo link works without special accounts
- [ ] "What we built today" is honest and specific
- [ ] Selected track matches the project
- [ ] Submitted with time to spare — the form takes a few minutes

Questions? Open a **[Question issue](../../../issues/new?template=question.yml)** or ask on Discord (https://discord.gg/jy3QDxQ3jK).
